# Create Contract Feature Plan

## Overview

Re-introduce the rental contract feature for the platform. This will allow the creation, viewing, and management of rental contracts between landlords (Party A) and tenants (Party B), conforming to the Vietnamese legal standard for room rentals. The feature includes a full-stack implementation spanning the Rust backend and Next.js frontend.

## Project Type

WEB

## Success Criteria

- [ ] Backend API endpoints exist to create and view contracts.
- [ ] Database schema properly supports contract data (dates, amounts, parties).
- [ ] Frontend allows users to create a contract with a form based on `hop-dong-thue-tro.html`.
- [ ] Frontend displays the contract details clearly.
- [ ] E2E or Unit tests verify contract creation logic.
- [ ] The feature is completely integrated without regressions.

## Tech Stack

- **Backend:** Loco (Rust) with SeaORM.
- **Frontend:** Next.js (React), Tailwind CSS, React Hook Form + Zod for validation.
- **Database:** PostgreSQL.
- **Testing:** Rust built-in tests (insta), Playwright/Jest for Frontend.

## File Structure

- `apps/api/server-api/src/models/_entities/contracts.rs` (Generated by SeaORM)
- `apps/api/server-api/src/controllers/contracts.rs`
- `apps/web/src/app/(dashboard)/contracts/page.tsx`
- `apps/web/src/app/(dashboard)/contracts/[id]/page.tsx`
- `apps/web/src/app/(dashboard)/contracts/create/page.tsx`
- `packages/shared/src/schemas/contracts.ts` (Shared types/Zod schemas)

## Task Breakdown

### 1. Database & Schema (P0)

- **Agent:** `database-architect`
- **Skills:** `database-design`
- **INPUT:** `docs/hop-dong-thue-tro.html` fields.
- **OUTPUT:** Loco migration generating `contracts` table and SeaORM entities.
- **VERIFY:** `cargo run -- db migrate` succeeds and `cargo test` runs without schema errors.

### 2. Backend Implementation (P1)

- **Agent:** `backend-specialist`
- **Skills:** `api-patterns`, `rust-pro`
- **INPUT:** Contract entity.
- **OUTPUT:** `POST /api/contracts` and `GET /api/contracts/:id` endpoints in `controllers/contracts.rs`.
- **VERIFY:** HTTP tests pass for both endpoints.

### 3. Shared Schemas & Types (P2)

- **Agent:** `frontend-specialist` (or shared)
- **Skills:** `clean-code`
- **INPUT:** Backend contract models.
- **OUTPUT:** Zod schemas and TypeScript types in `packages/shared`.
- **VERIFY:** Types compile (`npx tsc --noEmit`).

### 4. Frontend UI (P2)

- **Agent:** `frontend-specialist`
- **Skills:** `frontend-design`, `react-best-practices`
- **INPUT:** Shared types and API routes.
- **OUTPUT:** Next.js pages for listing, creating, and viewing contracts (`/contracts`, `/contracts/create`, `/contracts/[id]`).
- **VERIFY:** UI compiles and loads successfully without hydration errors.

### 5. Verification (P3)

- **Agent:** `test-engineer`
- **Skills:** `webapp-testing`
- **INPUT:** Completed feature.
- **OUTPUT:** Passing test suites.
- **VERIFY:** Run `python .agent/scripts/verify_all.py .`
