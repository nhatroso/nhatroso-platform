# DECISION LOG (ADR)

<aside>
🧠

Use ADRs to capture architecture and technology decisions. One decision per entry.

</aside>

### ADR template (copy for each decision)

- **ADR: [Short title]**
    - **Context**
        - What problem are we solving?
        - What constraints exist?
    - **Decision**
        - The chosen approach.
    - **Alternatives**
        - Option A
        - Option B
    - **Consequences**
        - Tradeoffs and risks.
    - **Date**
    - **Owner**

### Suggested naming

- ADR-001: Tech stack
- ADR-002: Database design approach
- ADR-003: Auth strategy
