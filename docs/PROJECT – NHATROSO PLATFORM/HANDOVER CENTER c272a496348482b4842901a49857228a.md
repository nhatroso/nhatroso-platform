### Handover notes

- **Tech Stack Shift**: Project migrated from planned NestJS/FastAPI to **Rust (Loco.rs)** for the API layer to gain performance and safety.
- **Data Model**: Uses **i32 serial IDs** (not UUID). Database supports dual SQLite (dev) / PostgreSQL (prod) via SeaORM.
- **Key Modules**:
  - `room_services`: Centralizes room vs service pricing logic.
  - `meter_requests`: Orchestrates monthly meter reading submissions between Tenant and Owner.
  - `invoices`: Flexible line-item generation via `invoice_details`.
- **Known Risks**: OCR descoped to manual entry for MVP; payment webhook integration is pending provider selection (VietQR recommended).
- **Communication**: Shared translations in `packages/translations` ensure consistent i18n across Web and Mobile.
