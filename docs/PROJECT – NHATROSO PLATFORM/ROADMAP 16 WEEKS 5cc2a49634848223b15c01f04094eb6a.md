### Roadmap overview

| Weeks | Phase              | Deliverable   | Status         |
| ----- | ------------------ | ------------- | -------------- |

| 5–8   | Sprint Development | Alpha         | 🔄 In Progress |

### Weekly milestones (lightweight)

- **Week 1:** ✅ Vision + problem statement, stakeholders, KPIs
- **Week 2:** ✅ BRD scope, out of scope, success metrics, approvals
- **Week 3:** ✅ SRS drafted and reviewed, MVP cut confirmed
- **Week 4:** ✅ SDD approved, definition of done agreed
- **Weeks 5–8:** 🔄 Sprint 1–3 execution (Alpha)
  - Sprint 1: Auth, property management (buildings/floors/rooms), services & pricing
  - Sprint 2: Contracts, meters, meter request workflow (mobile + web)
  - Sprint 3: Invoice management (create, calculate, void, pay), web dashboard polish
- **Weeks 9–12:** System testing, UAT, Beta release
- **Weeks 13–16:** Go-live, monitoring, incident readiness, handover

### Current status (April 2026)

**Completed features:**

- ✅ Monorepo setup (Turborepo + pnpm): `apps/api`, `apps/web`, `apps/mobile`, `packages/shared`, `packages/translations`
- ✅ API: Rust + Loco.rs + Axum + SeaORM; 15 controllers, 36 migrations
- ✅ Auth (JWT + refresh tokens), RBAC (Owner/Tenant)
- ✅ Property management: buildings, floors, rooms
- ✅ Services, room_services, price_rules (effective-dated)
- ✅ Contracts (1 active per room) + contract_tenants
- ✅ Meters + meter_readings + meter_request_configs + meter_requests
- ✅ Invoice lifecycle: create, calculate, void, pay, webhook
- ✅ File upload (`/static` mount)
- ✅ Owner web dashboard: 11 routes (buildings, floors, rooms, services, room-services, contracts, meters, meter-readings, meter-requests, meter-settings, invoices) — Next.js 16 + TailwindCSS v4 + next-intl (vi/en)
- ✅ Tenant mobile app: meter submission, history, request status — Expo 55 + NativeWind

**In progress / next:**
